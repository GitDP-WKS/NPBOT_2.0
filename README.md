# РЭС AI 2.0

Автономный сервис определения филиала и РЭС по адресу или тексту обращения для Республики Татарстан.

## Основные принципы

- Интерфейс Streamlit и API используют одно общее прикладное ядро.
- Все постоянные данные хранятся в PostgreSQL / Neon в таблицах `res_ai_v2_*`.
- Непротиворечивая запись источника не считается подтвержденной человеком.
- Уникальное название в одном адресном контексте получает расчетное доверие до 99,9%.
- Если одинаковое название встречается в двух разных контекстах, доверие без района составляет около 50%; в трех — около 33,3%.
- Район, СНТ и улица уточняют контекст и позволяют принять окончательное решение.
- Филиал всегда определяется по официальному РЭС.
- Администратор применяет решение сразу. Решение обычных проверяющих применяется после трех одинаковых голосов.
- Все действия, голоса, решения и запуски агента записываются в журнал.
- Модель обучается на однозначных адресах и размеченных реальных текстах. Конфликты и многозначные адреса исключаются.
- Новая модель не публикуется, если ухудшает общую точность, среднюю оценку по всем РЭС или качество отдельного РЭС.

## Как работает агент

```text
Excel / API / решение человека
            ↓
        событие в Neon
            ↓
   анализ измененной части
            ↓
автоматическое решение или задание человеку
            ↓
обновление обучающих данных
            ↓
подготовка кандидата модели после накопления решений
```

После загрузки файла агент автоматически анализирует только новые и связанные с ними адреса. Повторная загрузка того же файла не изменяет счетчики и не создает повторные задания.

Очередь событий защищена от повторной обработки, поддерживает повторные попытки после ошибок и восстановление прерванных запусков. Небольшая часть очереди обрабатывается автоматически при обычной работе пользователей. Полный контроль доступен администратору в разделе `Центр управления`.

## Запуск

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Streamlit Secrets

```toml
ADMIN_PASSWORD = "ваш-пароль"
DATABASE_URL = "postgresql://USER:PASSWORD@HOST/DATABASE?sslmode=require"
```

Без `DATABASE_URL` рабочее приложение не переходит молча на локальную базу. Локальная SQLite разрешена только для разработки и автоматических тестов.

## Первый запуск

1. Развернуть репозиторий в Streamlit Community Cloud.
2. Добавить `ADMIN_PASSWORD` и `DATABASE_URL` в Secrets.
3. Перезапустить приложение.
4. Войти как администратор.
5. Загрузить Excel в разделе `Загрузка`.
6. Проверить автоматически сформированные задания.
7. Открыть `Анализ и обучение`, подготовить кандидата и просмотреть показатели качества.
8. Опубликовать кандидата только после прохождения проверки качества.

- `Определение`.

- `Журнал`;
- `Настройки`.

## Программный интерфейс

```bash
uvicorn res_ai_v2.api:app --host 0.0.0.0 --port 8000
```

Основные методы:

- `GET /v1/health` — доступность сервиса;
- `GET /v1/system/status` — состояние базы и очереди агента;
- `POST /v1/predict` — определение филиала и РЭС;
- `GET /v1/tasks` — задания для проверяющего;
- `POST /v1/tasks/{id}/vote` — решение проверяющего;
- `POST /v1/files/excel` — загрузка Excel;
- `POST /v1/admin/agent/process` — ручная обработка очереди администратором.

Пример определения:

```json
{
  "text": "Лаишевский район, село Усады, нет электричества"
}
```

Административные методы требуют пароль в заголовке `X-Admin-Password`.
